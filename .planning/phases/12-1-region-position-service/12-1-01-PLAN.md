<objective>
Implement the Region service layer: typed helpers for list_regions(), set_region(), delete_region(), resolve_region(), resolve_all_regions(), validate_region_record(). Regions are stored in config["combat_regions_v2"] as canonical object records `{area, kind, threshold, enabled, label, migrated_from}`. Service mutates config and returns result; backend owns persistence (save_config). Fresh window rect on every resolve (D12.1-D02).
</objective>

<context>
Phase 12.0 migration established the canonical schema for combat_regions_v2:

```json
{
  "green_hp_bar": {
    "area": [0.30, 0.25, 0.70, 0.60],
    "enabled": true,
    "kind": "hsv_green",
    "threshold": 0.004,
    "migrated_from": "combat_regions"
  }
}
```

Phase 12.1 Plan 01 creates `src/services/region_service.py` following this object schema. Plan 02 creates position_service.py. Plan 03 wires both to backend commands and owns __init__.py. Service layer does NOT call save_config — backend command handler does after service returns.
</context>

## Task 2: Implement validate_region_record()

**read_first:** `src/utils/config.py` (lines 261-271, canonical schema fields)

**action:**
Implement `validate_region_record(name: str, record: Dict[str, Any]) -> tuple[bool, str | None]`:
1. Check `name` is non-empty string (strip, reject empty)
2. Check `record` is a dict
3. Check `record["area"]` exists and is list/tuple of length 4
4. Check each area value in [0.0, 1.0]: `0.0 <= v <= 1.0`
5. Check `x1 < x2` and `y1 < y2`
6. Check `(x2 - x1) * (y2 - y1) > 0`
7. Check `record["kind"]` exists and is a non-empty string (default to `"generic"` if absent)
8. Check `record["threshold"]` is None or a float >= 0 (coerce to float if string)
9. Check `record["enabled"]` is a bool (default to `True` if absent)
10. Return `(True, None)` on success, `(False, "error message")` on failure

Also implement `validate_area(area: List[float]) -> tuple[bool, str | None]` as a standalone helper that validates just the [x1,y1,x2,y2] list, used by set_region().

**acceptance_criteria:**
- `grep -n "def validate_region_record" src/services/region_service.py` finds the function
- `grep -n "def validate_area" src/services/region_service.py` finds the helper
- `grep -n "x1 < x2" src/services/region_service.py` confirms bbox validation

## Task 6: Implement resolve_region()

**read_first:** `src/utils/windows.py` (get_window_rect returns (left, top, right, bottom)), `src/utils/config.py` (lines 261-271, record["area"] access)

**action:**
Implement `resolve_region(config: dict, name: str) -> Optional[Dict[str, Any]]`:
1. Read `config["game_window_title"]` to find window
2. Call `get_window_rect(title)` — fresh call every time (D12.1-D02)
3. If rect is None, return `None`
4. Read `config["combat_regions_v2"][name]` — if not found, return `None`
5. Extract `area = record["area"]`
6. Convert normalized area [x1,y1,x2,y2] to pixels:
   - `width = right - left`, `height = bottom - top`
   - `abs_x1 = int(left + area[0] * width)`
   - `abs_y1 = int(top + area[1] * height)`
   - `abs_x2 = int(left + area[2] * width)`
   - `abs_y2 = int(top + area[3] * height)`
7. Return `{"name": name, "abs_area": [abs_x1, abs_y1, abs_x2, abs_y2], "area": area, "kind": record.get("kind"), "enabled": record.get("enabled", True)}`

**acceptance_criteria:**
- `grep -n "def resolve_region" src/services/region_service.py` finds the function
- `grep -n "get_window_rect" src/services/region_service.py` confirms fresh window rect
- `grep -n "area\[0\] \* width" src/services/region_service.py` confirms normalized-to-pixel conversion using area key
