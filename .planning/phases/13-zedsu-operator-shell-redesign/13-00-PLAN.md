<objective>

Create the CSS design system foundation for the operator shell: design tokens, base components, and shared CSS. This plan creates the `src/ZedsuFrontend/src/styles/` directory with `tokens.css`, `components.css`, and `app.css`. All subsequent plans import these files. The index.html shell and HUD both consume these tokens.

</objective>

<background>

Per D-01 (visual identity): tactical operator console aesthetic, no generic dark blue Tkinter. Per D-02 (typography): Space Grotesk + Geist Sans + JetBrains Mono. Per D-03 (motion): functional only, 160-220ms ease-out panel enter, 80ms button press. Per D-11 (anti-slop): no gradients, no nested cards, no bounce/elastic.

The design system establishes the foundation that 13-03 (shell layout) and 13-04 (Settings v3) depend on. This plan creates ONLY CSS files — no HTML structure or JS logic.

</background>

- `.planning/phases/13-zedsu-operator-shell-redesign/13-UI-SPEC.md` — Design tokens, spacing scale, color, typography, motion contract
