```python
yolo_detections = []
```

#### Step 4: Reposition/flee preempts M1 burst

File: `src/core/bot_engine.py`

In `_execute_engaged_combat()` (around line ~626), restructure the method so intent is checked BEFORE M1 clicks:

**Current (buggy):**
```python

    # equip melee...
    for _ in range(5):  # 5 M1 clicks ALWAYS run first
        pydirectinput.click()
        # ...
```

**Fixed:**
```python
def _execute_engaged_combat(self, action):
    intent = action.get("situation", {}).get("recommended_intent", "engage")

    # PHASE 12.5.1: If reposition or flee, skip M1 and move immediately
    if intent in ("flee", "reposition"):
        self.perform_dynamic_combat_movement(
            [keys_cfg.get("forward", "w"),
             keys_cfg.get("left", "a"),
             keys_cfg.get("backward", "s"),
             keys_cfg.get("right", "d")],
            bursts=1,
            situation=action.get("situation"),
            target_decision=action.get("_target_decision_obj"),  # Object, not dict
        )
        return

    # PHASE 12.5.1: Only M1 when intent is engage or pursue
    # equip melee...
    if not self.ensure_melee_equipped(slot1_key):
        pydirectinput.press(slot1_key)
        self.sleep(0.3)

    for _ in range(5):
        if not self.is_running():
            return
        pydirectinput.click()
        self.last_punch_time = time.time()
        if not self.sleep(random.uniform(0.06, 0.11)):
            return

        exit_result = self._check_combat_exit_conditions()
        if exit_result:
            return

    self.grant_provisional_melee(18)

    if random.random() < action.get("dodge_chance", 0.12):
        backward = keys_cfg.get("backward", "s")
        pydirectinput.keyDown(backward)
        self.sleep(random.uniform(0.15, 0.3))
        pydirectinput.keyUp(backward)

    # PHASE 12.5.1: Pass object for camera correction
    self.perform_dynamic_combat_movement(
        [keys_cfg.get("forward", "w"),
         keys_cfg.get("left", "a"),
         keys_cfg.get("backward", "s"),
         keys_cfg.get("right", "d")],
        bursts=random.randint(1, 2),
        situation=action.get("situation"),
        target_decision=action.get("_target_decision_obj"),  # Object
    )
```

#### Step 5: Fix dict/object handling in perform_dynamic_combat_movement

File: `src/core/bot_engine.py`

The `perform_dynamic_combat_movement()` method at line ~1293 already uses `getattr(target_decision, "center_error_x", 0.0)` — this is correct for objects but wrong for dicts (getattr on dict never returns the default; dicts don't raise AttributeError for missing keys, but they return the dict itself).

The fix in Step 4 ensures `target_decision` is always passed as the `TargetDecision` object (from `action["_target_decision_obj"]`). No code change needed in `perform_dynamic_combat_movement()` itself — the contract is now: **always pass object**.

However, add a defensive helper inside `perform_dynamic_combat_movement()` to handle any remaining dict inputs gracefully:

```python
def _get_center_error_x(target_decision, default=0.0):
    """Extract center_error_x regardless of whether input is object or dict."""
    if target_decision is None:
        return default
    if isinstance(target_decision, dict):
        return float(target_decision.get("center_error_x", default))
    return float(getattr(target_decision, "center_error_x", default))
```

Replace the `getattr(target_decision, "center_error_x", 0.0)` call at line ~1353 with `_get_center_error_x(target_decision)`.

#### Step 6: Reorder DeathClassifier priority

File: `src/core/death_classifier.py`

Current priority in `_classify()`:
```
1. crowd_death
2. combat_death  ← wrong — catches everything that zone_death would catch
3. zone_death
4. stuck_death (placeholder)
5. target_lost_death
6. unknown
```

**Fixed priority:**
```
1. crowd_death     — unambiguous (in_combat + crowd_risk >= 0.70 + visible >= 2)
2. zone_death     — edge_risk >= 0.70 (check BEFORE combat_death)
3. stuck_death    (placeholder, still commented out)
4. target_lost_death
5. combat_death   — catches all other combat deaths (including storm+combat edge cases)
6. unknown
```

In the `_classify()` method, move Rule 3 (zone_death) to be checked before Rule 2 (combat_death):

```python
# Rule 1: Crowd death (unchanged)
if in_combat and crowd_risk >= self.CROWD_RISK_DEATH_THRESHOLD and visible_enemy_count >= 2:
    breakdown["rule_matched"] = "crowd_death"
    confidence = min(1.0, 0.5 + crowd_risk * 0.5)
    return "crowd_death", confidence, breakdown

# PHASE 12.5.1: Rule 2 — Zone death (moved BEFORE combat_death)
# Zone deaths can happen even while in_combat, so check zone first
if edge_risk >= self.EDGE_RISK_DEATH_THRESHOLD:
    breakdown["rule_matched"] = "zone_death"
    confidence = 0.6
    return "zone_death", confidence, breakdown

# Rule 3: Combat death (shifted from Rule 2)
# Catches deaths that are purely combat-related, not zone or crowd
if in_combat or hit_confirmed:
    breakdown["rule_matched"] = "combat_death"
    confidence = 0.7 if in_combat else 0.5
    return "combat_death", confidence, breakdown

# Rule 4: Stuck death (placeholder — still commented out)
# Rule 5: Target lost death
if target_lost_ms >= self.TARGET_LOST_DEATH_THRESHOLD_MS:
    breakdown["rule_matched"] = "target_lost_death"
    confidence = 0.6
    return "target_lost_death", confidence, breakdown

# Rule 6: Unknown
breakdown["rule_matched"] = "unknown"
return "unknown", 0.3, breakdown
```
