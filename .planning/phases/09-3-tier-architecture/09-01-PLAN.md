```python
from typing import TypedDict, Protocol, Callable, Any, Optional
from enum import Enum

class LogLevel(str, Enum):
    INFO = "info"
    WARN = "warn"
    ERROR = "error"

class LogPayload(TypedDict):
    msg: str
    level: LogLevel

class StatusPayload(TypedDict):
    text: str
    color: str  # hex color like "#16a34a"

class DiscordPayload(TypedDict):
    message: str
    screenshot_path: Optional[str]
    event: str  # "match_end", "timeout", etc.

class ConfigFnReturns(TypedDict):
    config: dict  # full config dict
```

2. CoreCallbacks Protocol (per D-09a — Protocol + TypedDict):
```python
class CoreCallbacks(Protocol):
    """Interface for ZedsuCore → ZedsuBackend communication. Backend implements these."""

    def log(self, msg: str, level: LogLevel = LogLevel.INFO) -> None:
        """Log a message. Backend writes to file + optionally to frontend."""
        ...

    def status(self, text: str, color: str = "#475569") -> None:
        """Update status display. Backend broadcasts to frontend via /state."""
        ...

    def discord(self, message: str, screenshot_path: Optional[str] = None, event: str = "info") -> None:
        """Send Discord notification. Backend calls send_discord()."""
        ...

    def config(self) -> dict:
        """Return current config dict. Called every time bot reads config."""
        ...
```

3. No-Op fallback implementation:
```python
class NoOpCallbacks:
    """Default callbacks that do nothing. Use when ZedsuCore runs standalone without backend."""
    def log(self, msg, level="info"): pass
    def status(self, text, color="#475569"): pass
    def discord(self, message, screenshot_path=None, event="info"): pass
    def config(self): return {}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ZedsuCore → ZedsuBackend | CoreCallbacks interface — ZedsuCore trusts backend for config, logging, Discord |
| Config dict | Passed through callbacks — backend owns it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|------------|-----------------|
| T-09-01 | Tampering | Config dict passed via callbacks | mitigate | Backend owns config; Core only reads. No config writes from Core. |
| T-09-02 | Denial | Core callbacks block/timeout | mitigate | All callbacks are fire-and-forget (no return values checked in hot path). Backend handles errors internally. |
| T-09-03 | Info Disclosure | Config dict contains secrets (webhook URL) | accept | Local-only process; webhook URL not exposed to frontend HTTP API. Backend strips before /state. |
</threat_model>
