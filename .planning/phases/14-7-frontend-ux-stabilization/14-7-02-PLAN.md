# Plan: 14.7-02 — Overview Rebuild

**Status:** COMPLETE

## Changes Made

### `src/ZedsuFrontend/src/scripts/overview.js` (complete rewrite)

1. **Skeleton rendered ONCE** — `renderSkeleton()` generates all HTML on first load. Never called again.

2. **DOM refs cached** — After skeleton render, all element IDs are collected into a `refs` object. Polling updates use cached refs, not repeated `getElementById`.

3. **Selective DOM updates** — `renderOverview(state)` only updates `textContent` and `className` of existing elements. Zero `innerHTML` calls in the poll loop.

4. **Scroll position preserved** — `lastScrollTop` tracked via scroll listener. Restored after every update.

5. **Cleanup function** — `cleanupOverviewPage()` removes event listener and clears all refs. Exported and registered with page lifecycle.

6. **YOLO path truncation** — Model path shown as filename only (`yolo_gpo.onnx`). Full path in `title` attribute.

7. **New layout:**
   - Hero status strip: status dot, bot status, backend, combat state, kills, match
   - Hero actions: Start / Stop / E-Stop (wired to ShellApi)
   - Setup alert banner (hidden when no issues)
   - 4-card grid: System, Combat, Detection, Notifications
   - Quick links row

8. **Operational language** — No "Runtime", no "Current Match". Labels use operator terminology.
