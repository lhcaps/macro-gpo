# Plan: 14.7-01 — Router Lifecycle Fix

**Status:** COMPLETE

## Changes Made

### `src/ZedsuFrontend/src/scripts/shell.js`

1. **Router cleanup registry** — Added `registerPage()` / `unregisterPage()` calls from `page-lifecycle.js`. `navigateTo(page)` now calls `unregisterPage(currentPage)` BEFORE loading the new page. Previous page's cleanup function runs before the new page loads.

2. **`currentPage = null`** — Starts as `null` instead of `'overview'`. First navigation triggers normal load flow.

3. **Page module wrapper pattern** — Each `pageModules` entry now returns `{ load, cleanup }` instead of calling the loader directly. `navigateTo` calls `mod.load()` and registers `mod.cleanup` with the lifecycle manager.

4. **Overview cleanup** — `overview.js` now exports `cleanupOverviewPage()` that removes the `zedsu:state-update` listener. Shell calls it via `unregisterPage('overview')` when navigating away.

5. **Logs cleanup** — `logs.js` exports `cleanupLogsPage()` that clears the poll timer. Shell calls it when navigating away from logs.

6. **Discord cleanup** — `discord.js` exports `cleanupDiscordPage()` that removes event listeners.

### `src/ZedsuFrontend/src/scripts/ui/page-lifecycle.js` (NEW)

Created with:
- `registerPage(name, container, cleanup)` — stores cleanup function per page
- `unregisterPage(name)` — calls cleanup, removes from registry
- `getPageCleanup(name)` — access without unregistering
- `isPageActive(name)` — check if page is mounted

### `src/ZedsuFrontend/src/scripts/ui/dom.js` (NEW)

Created with:
- `$()` — cached `querySelector` with memoization
- `clearCache()` — clears cached refs (called on page navigation)
- `on()` / `off()` / `cleanupHandler()` — safe event binding with dedup
- `updateText()` / `updateClass()` — only updates if value changed
