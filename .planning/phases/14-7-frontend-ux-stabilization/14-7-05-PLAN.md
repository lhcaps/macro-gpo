# Plan: 14.7-05 — Logs Page Fix

**Status:** COMPLETE

## Changes Made

### `src/ZedsuFrontend/src/scripts/pages/logs.js`

1. **Empty state renders immediately** — `renderLogs()` always called after `fetchLogs()`. Empty array → `logs-empty-state` with "No logs yet" message. Spinner never hangs.

2. **Poll lifecycle tied to tab** — `startPolling()` / `stopPolling()` called on page enter/leave. Navigate away → `stopPolling()` clears `pollTimer`. Return to Logs → `startPolling()` restarts.

3. **Auto-scroll guard** — `_userScrolledUp` flag set when `scrollTop + clientHeight < scrollHeight - 50`. Auto-scroll only happens when user is at bottom.

4. **Open Folder** — Calls `ShellApi.sendCommand('open_logs_folder')` via backend. Falls back to toast `./logs/` if backend command not wired.

5. **Copy filtered logs** — Copies currently filtered log lines. Shows "No logs to copy" toast if empty.

6. **Cleanup function** — `cleanupLogsPage()` stops polling and clears state.

7. **`normalizeLogs()`** — String and object log entries normalized via `normalizeLogs()` from normalizers.js.

## Verification

- Open Logs tab with no backend running — "No logs yet" immediately, no spinner
- Open Logs tab with logs → leave tab → wait 30s → return — old logs still shown, new polling starts
- Scroll up to read old logs → new log arrives — stays scrolled up, does not jump
- Click Open Folder → backend command fired (or toast fallback)
