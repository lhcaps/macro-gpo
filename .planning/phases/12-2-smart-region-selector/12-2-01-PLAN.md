<objective>
Build the drag-to-select overlay module (`src/overlays/region_selector.py`) that lets the operator visually pick a screen region, normalized to [0-1] coordinates.

Purpose: Enables visual region definition without requiring manual coordinate entry. The overlay class provides non-blocking `run()` (starts a dedicated Tkinter thread) and blocking `get_result(timeout)` (waits for user confirmation via Event). Callers (the HTTP handler in the backend) receive the result synchronously after blocking.

Key constraint: Tkinter event.x/event.y are canvas-local coordinates (origin at top-left of the overlay window). Normalize directly: `norm_x = local_x / win_width`. Do NOT use absolute window coordinates for normalization.

Output: `src/overlays/region_selector.py` -- a self-contained overlay module with drag capture, local-coord normalization, and Event-based result delivery.
</objective>

Create `src/overlays/region_selector.py` with a `RegionSelectorOverlay` class implementing:

1. **Constructor `__init__(self, window_title: str, region_name: str = "combat_scan")`**:
   - Store window_title and region_name as instance attributes
   - Do NOT call get_window_rect() here -- call it fresh in run()
   - Initialize `self.result_data: dict | None = None`
   - Initialize `self.result_event = threading.Event()`

2. **`run()` -- Dedicated thread entry (Tkinter MUST run in its own thread)**:
   - Inside this method, create a new Tkinter `Tk()` root window
   - Call `self.root.withdraw()` to hide the root
   - Call `get_window_rect(self.window_title)` fresh -- if None, set `self.result_data = {'action': 'error', 'message': 'Window not found'}`, call `self.result_event.set()`, and return cleanly (do NOT raise)
   - Get window bounds: `left, top, right, bottom = rect`; `win_width = right - left`; `win_height = bottom - top`
   - Store as `self._win_left, self._win_top, self._win_width, self._win_height` for use in drag handlers
   - Create a `Toplevel(self.root)` as the overlay window
   - Set geometry: `f"{win_width}x{win_height}+{left}+{top}"`
   - `overrideredirect(True)` on the Toplevel (no title bar, no borders)
   - `attributes("-topmost", True)` on the Toplevel
   - `attributes("-alpha", 0.25)` on the Toplevel -- makes window transparent so game is visible through it
   - Set `bg` to a neutral color, e.g. `'#1a1a2e'`

3. **Canvas drag box**:
   - Create a Canvas with `bg='#1a1a2e'`, `cursor="crosshair"`, `width=win_width`, `height=win_height`
   - Pack the canvas to fill the Toplevel
   - Bind `<Button-1>` to `self._on_press` -- records `(event.x, event.y)` as start point, stores as `self._sx, self._sy`
   - Bind `<B1-Motion>` to `self._on_drag` -- redraws live rectangle
   - Bind `<ButtonRelease-1>` to `self._on_release` -- finalizes selection
   - Track `self._dragging = False` and `self._started = False`
   - Track `self._rect_id` (canvas item ID) and `self._label_id` (canvas item ID) for deletion

4. **Coordinate handling**:
   - Tkinter `event.x` and `event.y` are canvas-local coordinates (origin at top-left of the overlay window)
   - Clamp local coords to window bounds: `local_x = max(0, min(event.x, self._win_width))`, `local_y = max(0, min(event.y, self._win_height))`
   - Normalize using local coords: `norm_x = local_x / self._win_width`, `norm_y = local_y / self._win_height`
   - Do NOT use absolute screen coords for normalization

5. **Live visual feedback**:
   - On drag: delete old `self._rect_id` and `self._label_id` items
   - Draw new rectangle: `canvas.create_rectangle(sx, sy, ex, ey, outline='#FF9500', width=2, fill='', stipple='gray50')` -- stipple makes fill semi-transparent
   - Draw label: canvas rect bg `#1A1A1A` at bottom of selection + text `"{abs(ex-sx)} x {abs(ey-sy)}px"` in `("Segoe UI", 10)` white

6. **Key bindings**:
   - Bind `'<Return>'` to `self._confirm()`
   - Bind `'<Escape>'` to `self._cancel()`
   - Bind `'<Destroy>'` on Toplevel to `self._on_destroy` -- calls `self._cancel()` if result not yet set

7. **`_confirm()` -- Enter key handler**:
   - GUARD: `if self.result_event.is_set(): return` -- ignore if already confirmed or cancelled
   - If no drag completed (`not self._started`), do nothing (ignore Enter without a selection)
   - Compute normalized coords using LOCAL canvas coords:
     ```
     norm_x1 = self._sx / self._win_width
     norm_y1 = self._sy / self._win_height
     norm_x2 = self._ex / self._win_width
     norm_y2 = self._ey / self._win_height
     ```
   - Clamp each to [0.0, 1.0]
   - Set `self.result_data = {'action': 'confirm', 'area': [norm_x1, norm_y1, norm_x2, norm_y2], 'name': self.region_name}`
   - Set `self.result_event.set()`
   - Call `self.root.quit()` to exit the mainloop

8. **`_cancel()` -- Esc / close button handler**:
   - GUARD: `if self.result_event.is_set(): return` -- ignore if already confirmed
   - Set `self.result_data = {'action': 'cancel'}`
   - Set `self.result_event.set()`
   - Call `self.root.quit()` to exit the mainloop

9. **`_on_destroy()` -- Toplevel Destroy event**:
   - Only calls `self._cancel()` if result not yet set -- this prevents Destroy from overwriting a confirmed result

10. **`get_result(timeout=None)` -- Blocking accessor**:
    - `self.result_event.wait(timeout=timeout)` -- blocks until set or timeout
    - Returns `self.result_data` if set, else `None` on timeout
    - If `self.result_data` is already set, return it immediately (don't wait)

11. **Threading model**:
    - The `run()` method is designed to be called from a dedicated thread (created by the caller)
    - `run()` calls `self.root.mainloop()` which blocks until `self.root.quit()` is called
    - After `mainloop()` returns, the calling thread continues
    - For the HTTP handler pattern (Plan 02): HTTP handler thread creates overlay, starts dedicated non-daemon thread calling `run()`, blocks on `get_result()`, then processes the result

| T-12.2-04 | Information Disclosure | Region data exposed in logs | accept | Region coords are game UI coordinates, not user data |
