<objective>
Build the drag-to-select overlay module (`src/overlays/region_selector.py`) that lets the operator visually pick a screen region, normalized to [0-1] coordinates.

Purpose: Enables visual region definition without requiring manual coordinate entry. The overlay class provides non-blocking `run()` (starts a dedicated Tkinter thread) and blocking `get_result(timeout)` (waits for user confirmation via Event). Callers (the HTTP handler in the backend) receive the result synchronously after blocking on the Event.

Key threading constraint: Tkinter requires its own dedicated thread for the mainloop. The overlay runs in a non-daemon thread so the caller can block on it. The caller MUST NOT be the main thread.

Output: `src/overlays/region_selector.py` -- a self-contained overlay module with drag capture, normalization, and Event-based result delivery.
</objective>

Create `src/overlays/region_selector.py` with a `RegionSelectorOverlay` class implementing:

1. **Constructor `__init__(self, window_title: str, region_name: str = "combat_scan")`**:
   - Store window_title and region_name as instance attributes
   - Do NOT call get_window_rect() here -- call it fresh in run()
   - Initialize `self.result_data: dict | None = None`
   - Initialize `self.result_event = threading.Event()`

2. **`run()` -- Dedicated thread entry (Tkinter MUST run in its own thread)**:
   - Inside this method, create a new Tkinter `Tk()` root window (NOT Toplevel -- use `Tk()` for the root)
   - Call `self.root.withdraw()` to hide the root
   - Create a `Toplevel(self.root)` as the overlay window
   - Call `get_window_rect(self.window_title)` fresh -- if None, log error via `logging.getLogger()` at WARNING level, then return (do NOT raise)
   - Position the Toplevel over the window rect:
     - Set geometry: `f"{win_width}x{win_height}+{left}+{top}"`
     - Use `attributes('-topmost', True)` on the Toplevel
     - Use `overrideredirect(True)` on the Toplevel
     - Set bg to `'#000001'` (near-black, effectively invisible)
   - Set cursor to crosshair on the canvas

3. **Canvas drag box**:
   - Create a Canvas with `bg='#000001'`, `cursor="crosshair"`
   - Pack the canvas to fill the Toplevel
   - Bind `<Button-1>` to record start point `(self._sx, self._sy)`
   - Bind `<B1-Motion>` to draw live rectangle and update label
   - Bind `<ButtonRelease-1>` to finalize if size >= 5x5 pixels; otherwise discard and reset
   - Clamp all coordinates within window bounds: `max(left, min(x, right))` etc.

4. **Live visual feedback**:
   - On drag motion: delete old rectangle item, draw new one with outline `#FF9500`, width 2, no fill
   - Draw dimension label below the box: canvas rectangle bg `#1A1A1A`, text `"{w} x {h}px"` in `("Segoe UI", 10)` white
   - Delete and redraw label on every motion event

5. **Key bindings**:
   - Bind `'<Return>'` to `self._confirm()`
   - Bind `'<Escape>'` to `self._cancel()`
   - Bind `'<Destroy>'` on Toplevel to `self._cancel()`

6. **`_confirm()` -- Enter key handler**:
   - If no drag completed yet, do nothing (ignore Enter without a selection)
   - Compute normalized coords:
     ```
     norm_x1 = (x1 - left) / win_width
     norm_y1 = (y1 - top) / win_height
     norm_x2 = (x2 - left) / win_width
     norm_y2 = (y2 - top) / win_height
     ```
   - Clamp each to [0.0, 1.0]
   - Set `self.result_data = {'action': 'confirm', 'area': [norm_x1, norm_y1, norm_x2, norm_y2], 'name': self.region_name}`
   - Set `self.result_event.set()`
   - Call `self.root.quit()` to exit the mainloop

7. **`_cancel()` -- Esc / close button handler**:
   - Set `self.result_data = {'action': 'cancel'}`
   - Set `self.result_event.set()`
   - Call `self.root.quit()` to exit the mainloop

8. **`get_result(timeout=None)` -- Blocking accessor**:
   - `self.result_event.wait(timeout=timeout)` -- blocks until set or timeout
   - Returns `self.result_data` if set, else `None` on timeout
   - If `self.result_data` is not None, return it immediately (don't wait again)

9. **Threading model**:
   - The `run()` method is designed to be called from a dedicated thread (created by the caller)
   - `run()` calls `self.root.mainloop()` which blocks until `self.root.quit()` is called
   - After `mainloop()` returns, the calling thread (the overlay thread) can call `get_result()` -- but typically the caller blocks on `get_result()` from the HTTP handler thread
   - For the HTTP handler pattern (Plan 02): HTTP handler thread creates an overlay instance, starts a dedicated non-daemon thread calling `run()`, then blocks on `get_result()`. When Tkinter sets the event and calls `root.quit()`, the HTTP handler thread unblocks and processes the result.
