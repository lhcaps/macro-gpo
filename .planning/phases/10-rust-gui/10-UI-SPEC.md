# Phase 10 — UI Design Contract

> Visual and interaction contract for Phase 10: Modern Rust/Tauri GUI. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | **none** | Pure HTML/CSS/JS in Tauri WebView |
| Preset | **not applicable** | No component library |
| Component library | **none** | No React/Vue frameworks |
| Icon library | **emoji only** | Status icons via Unicode emoji (🔴⚡⏱⚔) |

**Rationale:** This is a minimal overlay HUD, not a full application UI. No Tailwind, no shadcn, no React. Pure HTML/CSS/JS embedded in Tauri WebView. JetBrains Mono locked from D-10b-04 and D-10c-04.

### Glow Effects

```css
/* Neon glow for COMBAT state */
text-shadow: 0 0 10px #ef4444, 0 0 20px #ef4444, 0 0 30px #ef4444;
box-shadow: 0 0 10px rgba(239, 68, 68, 0.3);

/* Pulse animation for SCANNING state */
@keyframes pulse {
  0%, 100% { opacity: 1; }
  50% { opacity: 0.6; }
}

/* Blink animation for ERROR state */
@keyframes blink {
  0%, 100% { opacity: 1; }
  50% { opacity: 0; }
}
```

### Accent Reserved For

- **Status state indicators only** — the colored glow/text that reflects FSM state
- **NOT** for buttons, links, or general UI elements (HUD has no interactive elements except via hotkeys)

### Destructive

- **Not applicable** — HUD is read-only display, no destructive actions in this UI

## Layout

### 2-Row Combat Focus Layout

```
┌────────────────────────────────────────┐
│  [ 🔴 COMBAT ]                         │  ← Top row: FSM state (20px bold)
│  ⏱ 05:23 | ⚔ Kills: 12 | ⚡ 15ms      │  ← Bottom row: stats (12px regular)
└────────────────────────────────────────┘
           300×80px, top-right corner
```

### Top Row (State Display)
- Large FSM state text with status emoji
- Format: `[ {emoji} {STATE_NAME} ]`
- Examples: `[ 🔴 COMBAT ]`, `[ 🔵 SCANNING ]`, `[ 🟢 POST_MATCH ]`
- Font: 20px JetBrains Mono, weight 700
- Color: varies by state (see status colors above)
- Glow: text-shadow based on state

### Bottom Row (Stats Display)
- Core runtime stats in compact format
- Format: `⏱ {elapsed} | ⚔ Kills: {kills} | ⚡ {detection_ms}ms`
- Examples: `⏱ 05:23 | ⚔ Kills: 12 | ⚡ 15ms`
- Font: 12px JetBrains Mono, weight 400
- Color: muted (`#9ca3af` gray)

### CSS Transitions

```css
.state-text {
  transition: color 0.3s ease, text-shadow 0.3s ease;
}
```

Color fades smoothly between states (e.g., blue SCANNING → red COMBAT).

## State Binding

### Backend State Contract

Poll `window.__TAURI__.core.invoke('get_backend_state')` every ~1 second.

| HUD Element | Backend Field | Format |
|-------------|---------------|--------|
| State emoji | `combat.combat_state` | Map FSM state → emoji |
| State text | `combat.combat_state` | `{STATE_NAME}` |
| State color | `status_color` | Hex from backend |
| Timer | local elapsed | `MM:SS` from start |
| Kills | `stats.kills` | `{n}` |
| Detection ms | local measured | `{n}ms` |

## Interaction Contract

### Hotkey Bindings (Fixed)

| Hotkey | Action | Implementation |
|--------|--------|----------------|
| F1 | Emergency Stop | `send_action('emergency_stop')` |
| F2 | Toggle HUD visibility | Show/hide overlay window |
| F3 | Start/Stop bot | `send_action('start')` / `send_action('stop')` |
| F4 | Open Settings | Reveal hidden main window |

**Note:** No hotkey editor UI. Fixed bindings per D-10d-03.

### JS IPC Commands

```javascript
// Poll state every ~1 second
const state = await window.__TAURI__.core.invoke('get_backend_state');

// Send hotkey actions
await window.__TAURI__.core.invoke('send_action', { action: 'start' });
await window.__TAURI__.core.invoke('send_action', { action: 'stop' });
await window.__TAURI__.core.invoke('send_action', { action: 'emergency_stop' });
```

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Initial state | `[ ○ IDLE ]` | App just started, not running |
| Timer format | `⏱ MM:SS` | Elapsed time since START pressed |
| Stats format | `⏱ {time} \| ⚔ Kills: {n} \| ⚡ {ms}ms` | Detection latency |
| Hidden main window | Settings accessible via F4 | Tkinter config stays (D-10e-01) |

**No empty state** — HUD is always showing something (IDLE at minimum).

**No error copy** — ERROR state shows `[ ❌ ERROR ]` with red blink.

**No destructive confirmation** — Emergency stop (F1) is immediate, no confirmation dialog.
