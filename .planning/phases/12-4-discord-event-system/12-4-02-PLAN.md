## 12.4-02 — FSM Event Wiring

Wire 5 Discord event types into the combat FSM via `callbacks.emit_event()`:

- `match_end` → `handle_post_match()` — replace legacy `callbacks.discord()` call
- `kill_milestone` → `CombatStateMachine.update()` — check thresholds on kill increment
- `combat_start` → `bot_loop()` / `auto_punch()` — emit on FSM SCANNING transition
- `death` → `BotEngine._emit_death_event_once()` — emit on death detection (guarded against double-send)
- `bot_error` → `bot_loop()` exception handler — emit on unhandled exception

### Task 4: death — Guarded Emit (Preventing Double-Send)

**Files modified:** `src/core/bot_engine.py`

**Key issue:** Smart path calls `on_death()` then immediately enters `handle_spectating_phase()` — both would fire death events without a guard. Non-smart path only enters `handle_spectating_phase()` directly. Solution: single `_death_event_sent` flag on BotEngine, consumed by a helper method.

**Action:**

1. In `BotEngine.__init__()` (already done in Task 1), confirm these two attributes are present:
   ```python
   self._death_event_sent = False
   ```

2. Add a helper method to `BotEngine` class (add near `clear_match_active()` or `mark_match_active()`):

   ```python
   def _emit_death_event_once(self, source: str):
       """
       Emit a single death Discord event, guarded against double-send.
       Smart path calls from CombatStateMachine.on_death() → _combat_sm.on_death() wrapper.
       Non-smart path calls from handle_spectating_phase() entry.

       Args:
           source: "combat_sm" or "spectating_phase" — for logging only
       """
       if self._death_event_sent:
           return
       self._death_event_sent = True
       try:
           if hasattr(self, '_callbacks') and self._callbacks:
               raw_region = self.get_search_region()
               region = None
               if raw_region:
                   region = {
                       "left": raw_region[0],
                       "top": raw_region[1],
                       "right": raw_region[2],
                       "bottom": raw_region[3],
                   }
               self._callbacks.emit_event(
                   "death",
                   title=f"You Died — Match #{self.app.match_count}",
                   message="You died",
                   match_id=self.app.match_count,
                   include_screenshot=True,
                   metadata={"screenshot_region": region},
               )
               self.log(f"[COMBAT] Death event dispatched (source: {source}).")
       except Exception:
           pass  # Death event failures do not interrupt spectating watch
   ```

3. Reset `_death_event_sent` in `BotEngine.mark_match_active()` — add at the start of the method:

   ```python

       if self.match_active:
           return False
       self._death_event_sent = False  # Phase 12.4: reset death guard on new match
       # ... rest of existing method ...
   ```

4. In `BotEngine.clear_match_active()`, add reset:
