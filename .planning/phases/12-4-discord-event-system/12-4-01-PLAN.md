## 12.4-01 — Worker & Multipart Fixes

Three blockers fixed in `src/services/discord_event_service.py` before wiring FSM events:

1. **Worker enqueued wrong type, didn't send.** Queue typed as `Queue[QueuedDiscordEvent | None]`. Worker now calls `send_discord_event(...)` with screenshot capture. `emit_event()` resolves webhook URL at emit time and enqueues `QueuedDiscordEvent`.

2. **bot_error sanitizer not wired.** `emit_event()` now calls `_sanitize_error_message()` before building the wrapped event, replacing URLs/credentials/paths/traceback refs with `[REDACTED]`.

3. **Multipart embed+file format wrong.** Screenshot path now uses `data={"payload_json": json.dumps(payload)}` instead of `json=payload` + `files={}`. Discord accepts both fields in multipart POST.
