Phase 14.7: Frontend UX Stabilization and Bridger-style Redesign

Root cause fixes:
- Router lifecycle: navigateTo() now calls page cleanup before loading new page.
  Fixes Overview listener leak bleeding into other tabs (Overview re-rendering
  shellMain when user was on Discord/Logs/etc.)
- Config normalization: normalizers.js prevents discord.events indexOf crash
  when backend returns non-array. asArray() returns [] for undefined/null/object.
- Logs spinner hang: renderLogs() called on every fetch including empty array.
  Empty state shown immediately, no hanging spinner.
- Toggle state: rollback on save failure, loading states, no page reload on save.
